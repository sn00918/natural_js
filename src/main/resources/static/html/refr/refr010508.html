Grid(N.grid)

    개요 Overview

    Grid(이하 N.grid)는 테이블(TABLE) 요소에 데이터를 바인드하여 테이블 타입으로 데이터를 표현해 주는 컴포넌트입니다. 헤더고정형, 리스트형의 두 가지 타입을 제공 합니다.

    그리드로 만들어질 TABLE 요소의 넓이는 고정(px등)이든 가변(%)이든 반드시 넓이가 지정 되어야 합니다.

    상세조회 팝업을 자동으로 생성 해 주는 링크 컬럼을 표시 하고 생성 된 컬럼의 헤더 컬럼에 선택 한 컬럼을 숨기고 숨김 해제 할 수 있는 기능과 버튼을 표시 합니다.

    true 를 지정 하면 전체 그리드에 헤더에 정의 되어 있는 컬럼들을 모두 보여주고 array 에 컬럼 명 들을 담아 인자로 지정 해 주면 지정 한 컬럼들만 보여 줍니다.

    컬럼들의 제목은 그리드 헤더의 제목 텍스트를 그대로 사용 합니다. 때문에 그리드 헤더에 정의 되어 있지 않은 컬럼은 표시 되지 않습니다. 그리드 헤더에 없는 컬럼을 상세조회 팝업에 표시 하려면 그리드에 컬럼으로 정의 하고 그리드를 초기화 한 다음 hide 메서드로 해당 컬럼을 숨기면 상세조회 팝업에서만 볼 수 있습니다.

    fixedcol 옵션에 의해 컬럼이 고정 된 상태에서는 컬럼 숨기기 / 보이기 기능이 정상적으로 작동하지 않을 수 있습니다.

    상세조회 팝업의 컬럼 타이틀 정보는 내부적으로 tbody 의 td 나 td 안에 있는 요소들의 id 값을 가져와 사용 합니다. tbody 의 td 안에 id 속성을 가진 요소가 2개 이상 이거나 헤더와 바디의 요소 구성이 많이 다를 경우는 컬럼 타이틀이 정상적으로 표시 되지 않을 수 있습니다. 이때는 thead 의 th 요소에 data-id 속성에 정렬 될 데이터 컬럼명을 정의 해 주면 정상적으로 작동 됩니다.

    data 관련 컴포넌트들 끼리 같은 데이터를 바인드 한 다음 데이터를 완벽하게 동기화 하기 위해서는 다음 사항들을 유의 하여 사용 바랍니다.

    Natural-JS 의 데이터 관련 컴포넌트들은 내부적으로 data 는 jQuery Object 화 하여 사용합니다.

    "컴포넌트인스턴스.bind(data)" 와 같이 data 를 bind 메서드의 인자로 입력 하면 data는 내부적으로 N(data) 와 같이 jQuery Object로 변환 되고 "N(data).컴포넌트" 와 같이 jQuery Object 화 되어 있는 데이터를 입력 하면 별다른 처리 없이 컴포넌트에서 사용 될 data 로 지정 됩니다.

    때문에 다음과 같이 data 를 따로 담아 놓고 공유 해서 사용하는 경우 컴포넌트의 add 메서드를 통해 새로운 데이터를 생성했을 때 따로 담아 둔 데이터가 동기화 되지 않는 문제가 발생 됩니다.

    
...
data : [{
	col01 : 1,
	col01 : 2,
}, {
	col01 : 3,
	col01 : 4,
}],
init : function() {
	var grid = N([]).grid();
	grid.bind(this.data);

	grid.data() === this.data //-> false

	// array 자체는 false 이지만 array 안의 요소들은 같은 참조로 연결됨.
	// 때문에 Form 에서 val 메서드로 데이터 수정 시 값이 동기화 됨.
	grid.data()[0] === this.data[0] //-> true

	// 하지만 다음과 같이 add 할 때는 동기화 되지 않음.
	grid.add();
	grid.data().length //-> 3
	this.length //-> 2
},
...

    data 인자의 type 이 number 타입이면 다음 옵션인 row 로 인식 됩니다.
    row object -

    새로운 데이터를 추가 할 행 인덱스를 지정 합니다. 지정한 행 인덱스 앞에 행 요소와 데이터가 추가 됩니다.
    remove - - N.grid 선택한 행 요소와 데이터를 삭제 합니다.
    row number|array - 제거할 행 인덱스를 지정 합니다. index 들을 array에 담아 인자로 지정 하면 한번에 여러 행을 제거 합니다.
    revert - - N.grid bind 했을 때의 초기 데이터나 add 시 생성된 초기 데이터 상태로 복원 합니다.
    row number|array -

    row index를 입력하면 해당 행 데이터만 복원 하고 입력하지 않으면 전체 행 데이터를 복원 합니다.

    index 들을 array에 담아 인자로 지정 하면 한번에 여러 행을 복원 합니다.
    validate - - N.grid 수정되거나 입력된 데이터 전체를 검증하여 검증결과를 반환 합니다. 검증 메시지는 입력요소 옆에 툴팁으로 표시 됩니다.
    row number - row index를 입력하면 해당 행데이터만 검증하고 입력하지 않으면 전체 데이터를 검증합니다.
    val - - value object|N.grid

    선택한 컬럼의 원래 값을 가져오거나 입력(수정) 합니다.

    val 인자를 입력하지 않으면 선택한 값을 반환하고 입력하면 선택한 키 컬럼의 값을 입력한 값으로 바꿔 줍니다.

    값 입력 시 입력한 키 에 해당하는 데이터 없으면 새로운 컬럼 데이터를 생성 합니다.
    row number - 가져올 값의 row index
    key string - 가져올 값의 key 컬럼명
    val string | number | boolean | null -

    저장할 값

    key 인자 값과 같은 id 속성값을 가진 HTML 요소가 있다면 null 은 빈 문자열 숫자는 숫자 문자열, Boolean 은 "true" 나 "false" 문자열로 변환 되어 저장 되고 대상 HTML 요소가 없다면 원형 그대로 저장 됩니다.
    select - - array

    select 나 multiselect 옵션 지정 시 그리드 안의 행을 명령으로 선택 할 수 있습니다.

    • 첫번째 인자(row)를 입력하지 않으면 선택 된 행 들의 행 index 들을 반환 하고 입력하면 입력 된 인덱스에 해당하는 행을 선택 합니다.
    row array|number -

    선택할 row index

    • 행 1개를 선택 하려면 인자를 number 타입 으로 지정해도 되고 array 타입 으로 지정해도 됩니다.
    • 행을 2개 이상 선택 하려면 인자를 array 타입 으로 지정해야 합니다.
    isAppend boolean -

    true 를 입력 하면 이전에 선택 된 행들을 선택 취소 하지 않고 추가로 선택 합니다. false 를 입력 하면 이전에 선택 된 행들을 모두 선택 취소 후 선택 합니다.
    check - - string

    checkAllTarget 과 checkSingleTarget 옵션으로 지정 한 체크박스 들을 명령으로 선택 할 수 있습니다.

    • 첫번째 인자(row)를 입력하지 않으면 선택 된 체크박스 가 속해 있는 행 index 들을 반환 하고 입력하면 입력 된 인덱스에 속해 있는 체크박스들을 선택 합니다.
    row array|number -

    체크할 체크박스들의 row index

    • 체크박스 1개를 선택 하려면 인자를 number 타입 으로 지정해도 되고 array 타입 으로 지정해도 됩니다.
    • 체크박스를 2개 이상 선택 하려면 인자를 array 타입 으로 지정해야 합니다.
    isAppend boolean -

    true 를 입력 하면 이전에 선택 된 체크박스 들을 선택 취소 하지 않고 추가로 선택 합니다. false 를 입력 하면 이전에 선택 된 체크박스 들을 모두 선택 취소 후 선택 합니다.
    hide - - N.grid

    선택한 컬럼을 숨겨 줍니다.

    다중 행 으로 TBODY 나 THEAD 가 구성 된 그리드는 정상 작동 하지 않을 수 있습니다.
    colIdxs number|array - 숨길 컬럼 인덱스를 지정 합니다. index 들을 array에 담아 인자로 지정 하면 한번에 여러 컬럼을 한번에 숨겨 줍니다.
    show - - N.grid

    숨겨진 컬럼을 보여 줍니다.

    다중 행 으로 TBODY 나 THEAD 가 구성 된 그리드는 정상 작동 하지 않을 수 있습니다.
    colIdxs number|array - 숨김 해제 할 컬럼 인덱스를 지정 합니다. index 들을 array에 담아 인자로 지정 하면 한번에 여러 컬럼을 한번에 숨김 해제 합니다.
    move - - N.grid

    선택한 행을 이동 합니다.
    fromRow number -

    이동 대상 행의 인덱스를 입력 합니다.
    toRow number -

    이동 할 위치의 행 인덱스를 입력 합니다.

    지정 한 행 이전으로 이동 됩니다. 마지막 행으로 이동 하려면 마지막 행 인덱스 값에 1을 더하여 입력 해 주세요.

    입력 한 값이 바인드 되어 있는 데이터의 전체 행 수 보다 클 경우 항상 마지막 행 다음으로 이동 됩니다.

    copy - - N.grid

    선택한 행을 복사 합니다.
    fromRow number -

    복사 대상 행의 인덱스를 입력 합니다.
    toRow number -

    복사 할 위치의 행 인덱스를 입력 합니다.

    지정 한 행 이전으로 복사 됩니다. 마지막 행으로 복사 하려면 마지막 행 인덱스 값에 1을 더하여 입력 해 주세요.

    입력 한 값이 바인드 되어 있는 데이터의 전체 행 수 보다 클 경우 항상 마지막 행 다음으로 복사 됩니다.

    5. onSelect 이벤트를 이용하지 않고 행 의 두번째 셀 클릭 시 동작하는 이벤트 핸들러를 jQuery on 을 이용하여 직접 정의.

    N.grid 의 행 요소들은 다시 바인드 하면 사라지고 다시 생성 되므로 bind나 click 대신 on 메서드를 이용하여 이벤트를 바인드 해야 합니다.

    
// Controller
N("selector").cont(function() {
    // 1. 그리드 바인드
	var grid = N(data).grid(N("selector"), view).bind();

    // 2. 행의 두번째 셀 클릭 시 동작하는 이벤트 핸들러 정의
	grid.context("tbody").on("click", "td:eq(1)", function() {

		// 3. 행 인덱스 얻어오기
		var rowEle = $(this).closest("tbody");
		var idx = grid.context("tbody").index(rowEle);
		var rowData = grid.data()[idx];

		// 4. 가져 온 행 인덱스와 행 요소, 행 데이터를 이용하여 로직을 처리
		N.log(idx, rowEle, rowData);
	});
});
