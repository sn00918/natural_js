List(N.list)

    개요 Overview

    List(이하 N.list)는 <ul> / <li> 요소에 데이터를 바인드하여 단일 컬럼 타입으로 데이터 목록을 표현 해 주는 컴포넌트입니다.

    API DEMO

    데모에는 모든 옵션 및 기능을 포함하고 있지 않습니다. 전체 기능 목록은 데모 바로 아래 있는 기능 문서들을 참고 바랍니다.

    The demo does not include all options and features. For a complete list of features, please refer to the function documentation directly below the demo.

    revert boolean false

    revert 기능을 활성화 하면 최초 바인딩된 데이터를 메모리에 적제 하므로 데이터 바인딩 성능이 저하될 수 있습니다.

    multiselect 옵션이 true 일 경우에는 unselect 옵션이 적용 되지 않습니다.

    전체 선택 만 해 주고 체크박스의 클릭 이벤트는 발생 시키지 않습니다.

    jQuery selector 문자열로 지정하며 selector 의 context는 리스트 body(li) 로 자동 지정 됩니다.

    checkAllTarget 대상 요소는 반드시 li 영역에 있어야 합니다.

    checkAllTarget 옵션과 checkSingleTarget 옵션은 둘중 하나만 사용 할 수 있습니다.

    id 속성을 지정하여 데이터를 바인드 하면 정상적으로 작동하지 않습니다. 선택 된 행의 인덱스를 가져오는 용도로만 사용 바랍니다.

    리스트의 행을 선택하기 위한 체크박스 요소들을 지정(단일 선택) 합니다.

    jQuery selector 문자열로 지정하며 selector 의 context는 리스트 body(li) 로 자동 지정 됩니다.

    checkSingleTarget 대상 요소는 반드시 li 영역에 있어야 합니다.

    checkAllTarget 옵션과 checkSingleTarget 옵션은 둘중 하나만 사용 할 수 있습니다.

    id 속성을 지정하여 데이터를 바인드 하면 정상적으로 작동하지 않습니다. 선택 된 행의 인덱스를 가져오는 용도로만 사용 바랍니다.

    헤더고정형 리스트에서는 스크롤페이징이 기본으로 적용됩니다. 페이징시 한번에 바인드하는 행수를 지정 합니다.

    너무 크게 지정하면 스크롤 페이징이 작동될때 브라우저에 부하가 발생되므로 리스트에 입력요소나 이미지가 들어 있다면 200 이하로 그렇지 않다면 1000 이하로 설정하는 것을 권장합니다.

    rowHandlerBeforeBind : function(rowIdx, rowEle, rowData) {
	// this : N.list 인스턴스
	// rowIdx : 생성되는 행의 index
	// rowEle : 생성되는 행의 row element(tbody)
	// rowData : 생성되는 행의 row data
}

    rowHandlerBeforeBind 는 행 마다 값이 바인드 되기 전 실행 됩니다.
    rowHandler function null

    리스트에서 data를 bind 하거나 add 할때 행이 렌더링 될 때 마다 발생 하는 이벤트 입니다. 이벤트 콜백 메서드의 인자로 생성될 당시의 row index, 행 요소, 로우 데이터를 넣어줍니다.

    rowHandler : function(rowIdx, rowEle, rowData) {
	// this : N.list 인스턴스
	// rowIdx : 생성되는 행의 index
	// rowEle : 생성되는 행의 row element(li)
	// rowData : 생성되는 행의 row data
}

    rowHandler 는 행 마다 값이 바인드 된 후 실행 됩니다.
    onSelect function null

    리스트의 행이 선택될때마다 발생되는 이벤트이다. 이벤트 콜백 메서드의 인자로 선택한 행의 row index, 행 요소, 로우 데이터, 바로전에 선택한 row index를 넣어준다.

    onSelect : function(rowIdx, rowEle, rowData, beforeRowIdx, e) {
	// this : N.list 인스턴스
	// rowIdx : 선택된 행의 index
	// rowEle : 선택된 행의 row element(li)
	// rowData : 선택된 행의 row data
	// beforeRowIdx : 이전 선택된 행의 index
	// e : Click event object
}

    select 옵션이 true 일 때만 발생 됩니다.

    unselect 옵션이 false(기본값) 이면 선택이 해제 되었을 때 rowIdx 인자 값에 -1을 반환 하고 선택 되었을 때는 선택 된 행의 인덱스를 반환 합니다. unselect 옵션이 true 이면 선택 해제 기능이 비활성화 되기 때문에 rowIdx 값은 항상 선택 된 행의 인덱스를 반환 합니다.

    onBind function null

    onBind : function(context, data, isFirstPage, isLastPage) {
	// this : N.list 인스턴스

	// data : 리스트에 바인드 된 데이터
    // isFirstPage : 스크롤 페이징 시 첫번째 페이지 여부(스크롤 페이징 하지 않으면 true 반환)
    // isLastPage : 스크롤 페이징 시 마지막 페이지 여부(스크롤 페이징 하지 않으면 true 반환)
}

    스크롤페이징에 의해 페이징 될 때도 해당 행수 만큼 페이지가 완성 될 때마다 발생합니다.

    Name Arguments Type Return Description
    data - - array[json object]|jquery object 컴포넌트에 바인드 되어 있는 최종 데이터를 반환 합니다.
    rowStatus boolean|string - 반환되는 데이터의 타입을 결정하거나 데이터를 필터링 하기 위한 구분자 입니다.

    rowStatus 인자 값에 따라 다음과 같은 타입의 데이터를 반환 합니다.

    • 입력하지 않았거나 undefined : array[json object] 타입의 가공 된 데이터를 반환 합니다.

    • false : 바인드 되어 있는 원형(jquery object 타입)의 데이터를 반환 합니다.

      다른 데이터 관련 컴포넌트와 양방향 데이터 바인딩을 가능하게 하려면 반드시 false 인자값을 넣어 줘야 합니다.

    • "checked" : checkAll 옵션과 checkAllTarget 옵션을 지정 했거나 checkOnlyTarget 옵션 지정 시 array[json object] 타입의 체크된 데이터를 반환 합니다.

    • "update" : array[json object] 타입의 수정된 데이터를 반환 합니다.
    • "delete" : array[json object] 타입의 삭제된 데이터를 반환 합니다.

    data 관련 컴포넌트들 끼리 같은 데이터를 바인드 한 다음 데이터를 완벽하게 동기화 하기 위해서는 다음 사항들을 유의 하여 사용 바랍니다.

    Natural-JS 의 데이터 관련 컴포넌트들은 내부적으로 data 는 jQuery Object 화 하여 사용합니다.

    "컴포넌트인스턴스.bind(data)" 와 같이 data 를 bind 메서드의 인자로 입력 하면 data는 내부적으로 N(data) 와 같이 jQuery Object로 변환 되고 "N(data).컴포넌트" 와 같이 jQuery Object 화 되어 있는 데이터를 입력 하면 별다른 처리 없이 컴포넌트에서 사용 될 data 로 지정 됩니다.

    때문에 다음과 같이 data 를 따로 담아 놓고 공유 해서 사용하는 경우 컴포넌트의 add 메서드를 통해 새로운 데이터를 생성했을 때 따로 담아 둔 데이터가 동기화 되지 않는 문제가 발생 됩니다.

    
...
data : [{
	col01 : 1,
	col01 : 2,
}, {
	col01 : 3,
	col01 : 4,
}],
init : function() {
	var list = N([]).list();
	list.bind(this.data);

	list.data() === this.data //-> false

	// array 자체는 false 이지만 array 안의 요소들은 같은 참조로 연결됨.
	// 때문에 Form 에서 val 메서드로 데이터 수정 시 값이 동기화 됨.
	list.data()[0] === this.data[0] //-> true

	// 하지만 다음과 같이 add 할 때는 동기화 되지 않음.
	list.add();
	list.data().length //-> 3
	this.length //-> 2
},
...

    위와 같은 문제를 해결하기 위해서 "컴포넌트인스턴스.bind(data)" 메서드로 바인드 할 데이터를 jQuery Object 화 하여 바인드 하면 데이터 연동 문제가 해결 됩니다.

    
...
data : [{
	col01 : 1,
	col01 : 2,
}, {
	col01 : 3,
	col01 : 4,
}],
init : function() {
	// 순수한 데이터 Array 를 jQuery Object 로 변환
	this.data = N(this.data);

	var list = N([]).list();
	list.bind(this.data);

	list.data() === this.data //-> true
	list.data()[0] === this.data[0] //-> true

	list.add();
	list.data().length //-> 3
	this.length //-> 3
},
...

    여러 컴포넌트에 같은 데이터를 바인드 하여 자동으로 데이터들이 연동 되도록 할 경우에는 데이터를 따로 변수에 담아놓고 사용하지 말고 데이터를 최초 바인드 한 컴포넌트에서 data() 메서드를 통해 data 를 꺼내여 연동 할 컴포넌트에 바인드하는 방법을 추천 합니다.

    
...
init : function() {
	// 아래 구문 추가
	this.data = N(this.data);

	// 컴포넌트 초기화
	var form = N([]).form();
	var list = N([]).list();

	// 데이터 조회
	N.comm("data url").submit(function(data) {
		// form 에 데이터 바인드
		form.bind(0, data);

		// list 에 form 의 데이터를 바인드
		list.bind(form.data());
	});

	// form.data() === list.data() //-> true
},
...
    data array[json object] - data인자를 지정하면 컴포넌트에 바인드 된 데이터가 지정한 데이터로 치환되어 바인드 됩니다.
    "append" string - 두번째 인자에 "append" 라는 문자열을 지정하면 처음 바인드 되어 있던 데이터에 다시 바인드 한 데이터를 합쳐 주고 첫번째 인자로 지정한 data를 리스트에 추가로 표시 해 줍니다.

    row object -

    remove - - N.list 선택한 행 요소와 데이터를 삭제 합니다.
    row number|array - 제거할 행 인덱스를 지정 합니다. index 들을 array에 담아 인자로 지정 하면 한번에 여러 행을 제거 합니다.
    revert - - N.list bind 했을 때의 초기 데이터나 add 시 생성 된 초기 데이터 상태로 복원 합니다.
    row number|array -

    row index를 입력하면 해당 행 데이터만 복원 하고 입력하지 않으면 전체 행 데이터를 복원 합니다.

    index 들을 array에 담아 인자로 지정 하면 한번에 여러 행을 복원 합니다.
    validate - - N.list 수정되거나 입력된 데이터 전체를 검증하여 검증결과를 반환 합니다. 검증 메시지는 입력요소 옆에 툴팁으로 표시 됩니다.
    row number - row index를 입력하면 해당 행데이터만 검증하고 입력하지 않으면 전체 데이터를 검증합니다.
    val - - value object|N.list

    선택한 컬럼의 원래 값을 가져오거나 입력(수정) 합니다.

    val 인자를 입력하지 않으면 선택한 값을 반환하고 입력하면 선택한 키 컬럼의 값을 입력한 값으로 바꿔 줍니다.

    값 입력 시 입력한 키 에 해당하는 데이터 없으면 새로운 컬럼 데이터를 생성 합니다.

    key string - 가져올 값의 key 컬럼명

    key 인자 값과 같은 id 속성값을 가진 HTML 요소가 있다면 null 은 빈 문자열 숫자는 숫자 문자열, Boolean 은 "true" 나 "false" 문자열로 변환 되어 저장 되고 대상 HTML 요소가 없다면 원형 그대로 저장 됩니다.

<script type="text/javascript">
	N(data).list({

    5. onSelect 이벤트를 이용하지 않고 행 의 특정 요소 클릭 시 동작하는 이벤트 핸들러를 jQuery on 을 이용하여 직접 정의.

    N.list 의 행 요소들은 다시 바인드 하면 사라지고 다시 생성 되므로 bind나 click 대신 on 메서드를 이용하여 이벤트를 바인드 해야 합니다.

    
// Controller
N("selector").cont(function() {
    // 1. 리스트 바인드
	var list = N(data).list(N("selector"), view).bind();

    // 2. 행의 특정 요소(#col01) 클릭 시 동작하는 이벤트 핸들러 정의
	list.context("li").on("click", "#col01", function() {

		// 3. 행 인덱스 얻어오기
		var rowEle = $(this).closest("li");
		var idx = list.context("li").index(rowEle);
		var rowData = list.data()[idx];

		// 4. 가져 온 행 인덱스와 행 요소, 행 데이터를 이용하여 로직을 처리
		N.log(idx, rowEle, rowData);
	});
});
