Documents(N.docs)

    개요 Overview

    Documents(이하 N.docs) 는 Natural-JS 로 제작 된 메뉴 페이지들을 MDI(Multi Document Interface) 나 SDI(Single Document Interface) 타입으로 표시 할 수 있는 페이지 컨테이너 입니다.

    생성자 Constructor

    Name Arguments Type Return Description
    N.docs - - N.docs N.docs의 객체 인스턴스를 생성합니다.
    context jquery object - 기본옵션의 context 옵션과 같습니다.
    opts object - 기본 옵션을 object 타입으로 지정 합니다.
    N(obj).docs - - N.docs

    N.docs의 객체 인스턴스를 jQuery 플러그인 방식으로 생성합니다.

    N.docs과 동작되는 로직은 같으며 내부적으로 N()의 첫번째 인자가 N.docs()의 첫 번째 인자로 지정 됩니다.
    opts object - N.docs의 opts 옵션과 같습니다.

    기본옵션 Default Options

    Name Type Default Required Description
    context jquery object null ○ Documents컴포넌트화 할 Documents 요소를 지정합니다.
    multi boolean true

    MDI, SDI 여부

    true 이면 MDI 타입으로 탭 링크들과 컨텐츠들을 표시하고 false 이면 컨텐츠 탭 링크는 표시 하지 않고 단 한 개의 컨텐츠만 표시 합니다.
    maxStateful number 0

    MDI 방식으로 사용할 때 탭 컨텐츠가 열릴수록 웹 브라우저의 속도가 느려지는 현상을 방지하기 위해 최대 상태유지 컨텐츠 개수를 설정 할 수 있습니다.

    새로운 컨텐츠를 열때 열려있는 탭 컨텐츠의 개수가 최대 상태유지 탭 개수를 초과 하면 먼저 열린 순서대로 컨텐츠의 내용을 제거해서 해당 컨텐츠의 마지막 상태(입력이나 조회 값등)가 사라지게 되고 상태유지가 제거 된 탭 링크 클릭 시 해당 컨텐츠의 초기 화면이 다시 불러와 집니다.

    maxTabs number 0

    MDI 방식으로 사용할 때 탭 컨텐츠가 열릴수록 웹 브라우저의 속도가 느려지는 현상을 방지하기 위해 최대 탭 컨텐츠 개수를 설정 할 수 있습니다.

    새로운 컨텐츠를 열때 열려있는 탭 컨텐츠의 개수가 최대 탭 개수를 초과 하면 더이상 탭 컨텐츠를 열 수 없습니다. 새로운 컨텐츠를 열려면 기존 탭 중에 하나를 닫아야 합니다.

    addLast boolean false

    add 메서드를 호출 해서 새로운 페이지가 추가 되었을 때 탭을 뒤로 추가 할 것인 지 여부
    tabScroll boolean false

    탭들을 마우스 드래그나 터치로 스크롤 하는 기능 활성화 여부

    사이트의 CSS 적용 방식에 따라 마지막 탭이 잘리거나 여백이 생길 수 있습니다. 이때는 다음 옵션값을 조정하여 정상적으로 표시 할 수 있습니다. 

    N("#tab").tab({
	tabScrollCorrection : {
		// 마지막 탭이 잘리거나 여백이 생길때
		// 이 값을 1씩 증가시키거나 빼서 조정 해 주세요.
		rightCorrectionPx : 0
	}
});

    natural.config.js 의 N.context.attr("ui.shell").docs 에 위 옵션을 기입하면 N.docs 컴포넌트 전체에 적용 됩니다.
    closeAllRedirectURL string null

    "전체 닫기" 버튼을 눌렀을 때 closeAllRedirectURL 옵션 값이 null 이면 활성 화 된 탭을 남겨 두고 나머지 탭들을 닫아 주고 url 문자열을 입력 하면 해당 url 로 리다이렉트 됩니다.

    SPA(Single Page Application) 방식으로 사이트를 개발 할 때는 메인(홈) 화면의 url 을 입력 하여 브라우저의 가비지 리소스들을 정리하기 위해 페이지 리다이렉션을 유도 하는것이 좋습니다.
    entireLoadIndicator boolean false

    컨텐츠와 컨텐츠의 데이터등이 로딩 되는 동안 로딩 상태 바를 보여 줄 것인 지 여부
    entireLoadScreenBlock boolean false

    컨텐츠와 컨텐츠의 데이터등이 로딩 되는 동안 컨텐츠 화면을 차단 할 것인 지 여부
    entireLoadExcludeURLs array []

    entireLoad 관련 이벤트나 옵션에서 캡쳐 되는 요청 중 entireLoadExcludeURLs 로 지정 한 URL 들은 캡쳐에서 제외 합니다.

    등록 된 URL 들은 entireLoadIndicator 옵션과 entireLoadScreenBlock 옵션에도 연동 되어 표시 됩니다.
    alwaysOnTop boolean false "메뉴목록" 대화상자를 항상 최 상위에 표시 할건 지 여부
    alwaysOnTopCalcTarget string "div, span, ul, p, nav, article, section" alwaysOnTop 옵션 적용 시 최 상위 z-index를 계산하기 위한 대상 요소지정

    jQuery selector 문자열로 지정 합니다

    Documents.request

    Document.request(N.docs.request) 는 단독으로 사용될 수 없고 N.docs 로 컨텐츠를 열때 마다 생성 되는 모듈입니다. request는 요청에 대한 옵션을 지정 할 수 있고 요청정보를 가지고 있고 페이지간 데이터 전달을 위한 attr 메서드가 구현 되어 있습니다.

    add 나 reload 함수를 호출 하기 전에 다음과 같이 request attribute 를 정의 하면

    
var docs = N("container selector").docs();
docs.request.attr("pageParam", { a : 1 }).add("page id", "page name", { url : "url" });
// docs.request.attr("pageParam", { a : 1 }).reload();

    불러와진 컨텐츠의 Controller(N.cont) 오브젝트의 init 함수의 두번째 인자(request)로 불러 올 수 있습니다.

    
N("page id").cont({
	init : function(view, request) {
		N.log(request.attr("pageParam")); // --> { a : 1 }
		// request 는 다른 함수들에서 this.request 로 참조 가능
	}
})

    Documents.request 의 자세한 스펙과 사용법은 Communicator.request 모듈을 그대로 사용하므로 [Natural-ARCHITECTURE] 메뉴의 [Communicator 탭] 에서 Communicator.request 섹션을 참고 하세요.

    함수 Methods

    Name Arguments Type Return Description
    context - - jquery object 컴포넌트를 적용한 요소를 반환 합니다.
    selector string - selector 문자열을 입력하면 context 옵션으로 지정한 요소에서 선택된 요소를 반환 합니다.
    add - - N.docs 새로운 컨텐츠과 컨텐츠 탭을 추가 합니다.
    docId string -

    컨텐츠(페이지) 아이디를 입력 합니다.

    이미 같은 아이디로 지정 된 컨텐츠가 있다면 해당 컨텐츠가 활성화 됩니다.
    docNm string - 컨텐츠 명(페이지 명)을 입력 합니다.
    docOpts object -

    컨테츠 옵션(페이지 옵션)을 입력 합니다.

    
var docOpts = {
	url : "url", // 컨텐츠(페이지) URL
	docId : docId, // 컨텐츠(페이지) 아이디
	docNm : docNm, // 컨텐츠(페이지) 명
	onBeforeLoad : function(docId, target) {
		// 추가 되는 컨텐츠(페이지) 에만 적용되는 onBeforeLoad 이벤트.

	onLoad : function(docId) {

		// 추가 되는 컨텐츠(페이지) 에만 적용되는 onBeforeActive 이벤트.

	onActive : function(docId) {

	},
	onBeforeInactive : function(docId) {
		// 추가 되는 컨텐츠(페이지) 에만 적용되는 onBeforeInactive 이벤트.
	},
	onInactive : function(docId) {
		// 추가 되는 컨텐츠(페이지) 에만 적용되는 onInactive 이벤트.
	},
	onBeforeRemoveState : function(docId) {
		// 추가 되는 컨텐츠(페이지) 에만 적용되는 onBeforeRemoveState 이벤트.
	},

	},
	onBeforeRemove : function(docId) {
		// 추가 되는 컨텐츠(페이지) 에만 적용되는 onBeforeRemove 이벤트.

    개별 이벤트들은 전역 이벤트가 실행 된 후 실행 됩니다.
    active - - N.docs 선택한 컨텐츠과 컨텐츠 탭을 활성화 합니다.
    docId string - 컨텐츠(페이지) 아이디를 입력 합니다.
    removeState - - N.docs 선택한 컨텐츠의 상태를 제거 합니다.
    docId string - 컨텐츠의 상태를 제거할 컨텐츠(페이지) 아이디를 입력 합니다.
    remove - - N.docs 선택한 컨텐츠과 컨텐츠 탭을 제거 합니다.
    docId string - 컨텐츠(페이지) 아이디를 입력 합니다.
    unconditional boolean - true 를 입력 하면 수정 중인 입력이 있더라도 무조건 제거 합니다.
    doc - - object|array

    선택한 컨텐츠의 정보(docOpts)를 가져 옵니다.
    docId string -

    정보를 가져 올 컨텐츠(페이지) 아이디를 입력 합니다.

    입력하지 않으면 모든 컨텐츠의 정보를 array 로 반환 합니다.
    cont - - object

    선택한 컨텐츠의 Controller(N.cont) 오브젝트를 가져 옵니다.
    docId string -

    Controller를 가져 올 컨텐츠(페이지) 아이디를 입력 합니다.
    reload - - N.docs

    선택한 컨텐츠(페이지)를 리로드 합니다.

    reload 함수 호출 전에 request.attr 함수로 값을 지정하지 않았다면 리로드 전의 request 객체의 값은 리로드 해도 유지 됩니다.
    docId string -

    리로드 할 컨텐츠(페이지) 아이디를 입력 합니다.
    callback string -

    리로드가 완료 되면 실행 되는 콜백을 지정 합니다. 필요 없으면 지정 하지 않아도 됩니다.

    콜백의 인자로 html 과 Communicator.request 객체가 반환 됩니다.

    
docsInstance.reload("docId", function(html, request) {
	N.log(html, request);
});

    예제 Examples

    1. Documents 생성 및 페이지 추가

    
<section id="docsContainer"></section>

<script type="text/javascript">
   	var docs = N("#docsContainer").docs();
   	docs.add("ex", "예제", { url : "ex.html" });
</script>

    2. 페이지에 파라미터 전달

    
<section id="docsContainer"></section>

<script type="text/javascript">
   	var docs = N("#docsContainer").docs();
   	docs.request.attr("a", 1).request.attr("b", 2).add("ex", "예제", { url : "ex.html" });

   	// ex.html 의 N.cont 오브젝트의 init 함수의 두번째 인자인 request 객체에서
   	// request.attr("a");
   	// request.attr("b");
   	// 로 받을 수 있음.
</script>
