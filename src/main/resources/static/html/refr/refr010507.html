Form(N.form)

    개요 Overview

    Form(이하 N.form)은 폼 입력, 조회 양식을 구성하는 폼 블록(Block)요소(DIV, TABLE, SPAN 등)에 데이터를 바인드(bind)하거나 생성(add)하여 폼을 구현 해 주는 컴포넌트입니다.

    N.form 의 context 요소 안에 다시 N.form 이나 데이터를 바인드하는 컴포넌트(N.grid, N.list 등)의 context 를 지정 했을 때 상위 N.form 의 context 요소에 포함 된 요소들의 id 속성 값과 하위 N.form 의 context 요소에 포함된 요소들의 id 속성 값이 중복 되면 절대 안 됩니다. N.form 의 context 요소들을 중복 되지 않게 나누어 지정 하던 지 data 의 컬럼들을 중복 되지 않게 나누어 바인드 해야 합니다. 멀티 폼 바인드에 대한 예제는 [예제] > [멀티 폼 바인드] 메뉴 에서 확인 바랍니다.

    신규입력이나 검색 블록 등 데이터를 신규로 생성해야하는 폼 등은 add 메서드를 호출하여 폼 안의 입력요소들을 기본으로 데이터를 생성하고 수정이나 조회 등 데이터를 요소들에 바인드 해야 하는 폼은 bind 메서드를 호출 해서 사용 해야 합니다. bind 메서드와 add 메서드는 폼의 용도에 따라 반드시 구분하여 사용되야 하며 폼을 add 한 후 같은 폼 인스턴스로 생성 된 row 에 다시 bind 하면 폼이 정상적으로 작동하지 않을 수 있습니다. add 하면서 기본값을 지정 해야 한다면 add 의 인자로 기본값이 될 data를 넣어 주세요. 어쩔수 없이 add 나 bind 후 같은 폼요소에 다시 bind 해야 한 다면 add 나 bind 후 unbind 를 호출 한 다음 bind 를 호출 하거나 폼 초기화 시 autoUnbind 옵션을 활성화 시켜 주세요.

    입력요소의 값을 변경하거나 val 메서드로 데이터를 변경하면 rowStatus 프로퍼티가 생성되고, 입력은 "insert", 수정은 "update", 삭제는 "delete" 값이 입력 됩니다.

    단위(컬럼) 요소에 값이 바인드 되기전 실행 되는 이벤트입니다.

    natural.config.js 에 전역으로 설정 시 N.form 을 이용하는 모든 컴포넌트(N.grid, N.list)들에도 적용되어 해당 컴포넌트에서 값을 바인드 할 때도 N.form 의 전역 onBeforeBindValue 이벤트 핸들러가 실행 됩니다. 이것을 활용해 바인드 되는 모든 값들과 요소들을 한번에 가공 또는 조작 할 수 있습니다.

    onBeforeBindValue 는 natural.config.js 에 전역으로 설정 시 컴포넌트에 정의 한 이벤트핸들러의 결과값이 전역 이벤트의 값(val) 인자로 지정 됩니다. 때문에 다른 이벤트 들 처럼 전역 으로 설정한 이벤트 핸들러를 컴포넌트 이벤트에서 중단 할 수 없습니다.

    onBeforeBind : function(ele, val, action) {
    // this : N.form 인스턴스
    // ele : 바인드 될 요소
    // val : 바인드 될 값
    // action : 호출 함수 명 - "bind" | "val"
    return val; // 반드시 val 을 (가공 후) 다시 리턴 해야 함.
}
    onBeforeBind function null

    폼에 데이터가 바인드 되기 전 발생 되는 이벤트입니다.

    natural.config.js 에 전역으로 설정 시 N.form 을 이용하는 컴포넌트(N.grid, N.list)들에 값을 바인드 할 때도 모두 적용 됩니다.

    onBeforeBind : function(context, rowData) {
	// this : N.form 인스턴스
	// context : 폼 context 요소
	// rowData : 폼에 바인드 될 데이터
}

    폼에 데이터가 바인드가 완료 된 후 발생 되는 이벤트입니다.

    natural.config.js 에 전역으로 설정 시 N.form 을 이용하는 컴포넌트(N.grid, N.list)들에 값을 바인드 할 때도 모두 적용 됩니다.

	// this : N.form 인스턴스

}

    data 관련 컴포넌트들 끼리 같은 데이터를 바인드 한 다음 데이터를 완벽하게 동기화 하기 위해서는 다음 사항들을 유의 하여 사용 바랍니다.

    Natural-JS 의 데이터 관련 컴포넌트들은 내부적으로 data 는 jQuery Object 화 하여 사용합니다.

    "컴포넌트인스턴스.bind(data)" 와 같이 data 를 bind 메서드의 인자로 입력 하면 data는 내부적으로 N(data) 와 같이 jQuery Object로 변환 되고 "N(data).컴포넌트" 와 같이 jQuery Object 화 되어 있는 데이터를 입력 하면 별다른 처리 없이 컴포넌트에서 사용 될 data 로 지정 됩니다.

    때문에 다음과 같이 data 를 Controller object 에 따로 담아 놓고 공유 해서 사용하는 경우 컴포넌트의 add 메서드를 통해 새로운 데이터를 생성했을 때 따로 담아 둔 데이터가 동기화 되지 않는 문제가 발생 됩니다.

    
...
data : [{
	col01 : 1,
	col01 : 2,
}, {
	col01 : 3,
	col01 : 4,
}],
init : function() {
	var form = N([]).form();
	form.bind(0, this.data);

	form.data() === this.data //-> false

	// array 자체는 false 이지만 array 안의 요소들은 같은 참조로 연결됨.
	// 때문에 Form 에서 val 메서드로 데이터 수정 시 값이 동기화 됨.
	form.data()[0] === this.data[0] //-> true

	// 하지만 다음과 같이 add 할 때는 동기화 되지 않음.
	form.add();
	form.data().length //-> 3
	this.length //-> 2
},
...

    위와 같은 문제를 해결하기 위해서 "컴포넌트인스턴스.bind(row, data)" 메서드로 바인드 할 데이터를 jQuery Object 화 하여 바인드 하면 데이터 연동 문제가 해결 됩니다.

    
...
data : [{
	col01 : 1,
	col01 : 2,
}, {
	col01 : 3,
	col01 : 4,
}],
init : function() {
	// 순수한 데이터 Array 를 jQuery Object 로 변환
	this.data = N(this.data);

	var form = N([]).form();
	form.bind(0, this.data);

	form.data() === this.data //-> true
	form.data()[0] === this.data[0] //-> true

	form.add();
	form.data().length //-> 3
	this.length //-> 3
},
...

    여러 컴포넌트에 같은 데이터를 바인드 하여 자동으로 데이터들이 연동 되도록 할 경우에는 데이터를 따로 변수에 담아놓고 사용하지 말고 데이터를 최초 바인드 한 컴포넌트에서 data() 메서드를 통해 data 를 꺼내어 연동 할 컴포넌트에 바인드하는 방법을 추천 합니다.

    
...
init : function() {
	// 아래 구문 추가
	this.data = N(this.data);

	// 컴포넌트 초기화
	var form = N([]).form();
	var grid = N([]).grid();

	// 데이터 조회
	N.comm("data url").submit(function(data) {
		// form 에 데이터 바인드
		form.bind(0, data);

		// grid 에 form 의 데이터를 바인드
		// grid 와 form 의 데이터가 양방향 바인딩이 되기 위해서는 form.data() 함수 호출 시
		// false 인자를 지정 하여 데이터 원형이 반환 되도록 해야 합니다.
		grid.bind(form.data(false));
	});

	// form.data() === grid.data() //-> true
},
...

    unbind - - N.form

    N.form 의 context로 지정한 블록 요소의 bind 관련 이벤트나 상태, 스타일등을 제거하여 bind 하기 전 상태로 되돌려 줍니다.

    폼에 바인드 되어있는 data는 제거 되지 않고 this.options.row 변수 값도 그대로 유지 됩니다.

    새로운 로우 데이터를 생성 합니다.

    로우 데이터 생성 시 폼요소안의 입력요소의 id 와 값들로 데이터를 생성합니다.

    신규입력 페이지와 같이 입력된 데이터를 서버에 보낼 때 사용 하면 됩니다.

    N.form 초기화 후 add 메서를 호출하기만 하면 폼요소의 값들이 새로 생성된 데이터와 연동되므로 저장 버튼에 data 메서드를 호출하여 서버로 보내기만 하면 됩니다.
    data object - 새로운 행을 생성할 때 컬럼값들의 기본값을 지정 할 수 있습니다.

    인자로 지정된 데이터는 add 호출 시 생성된 기본 데이터를 확장 합니다.

    data 인자의 type 이 number 타입이면 다음 옵션인 row 로 인식 됩니다.
    row object -

    새로운 데이터를 추가 할 내부 데이터의 행 인덱스를 지정 합니다. 지정한 행 인덱스 앞에 데이터가 추가 됩니다.
    revert - - N.form bind 했을 때의 초기 데이터나 add 시 생성 된 초기 데이터 상태로 복원 합니다.
    validate - - N.form 수정되거나 입력된 데이터 전체를 검증하여 검증결과를 반환 합니다. 검증 메시지는 입력요소 옆에 툴팁으로 표시 됩니다.
    val - - value object|N.form

    선택한 컬럼의 원래 값을 가져오거나 입력(수정) 합니다.

    val 인자를 입력하지 않으면 선택한 값을 반환하고 입력하면 선택한 키 컬럼의 값을 입력한 값으로 바꿔 줍니다.

    값 입력 시 입력한 키 에 해당하는 데이터 없으면 새로운 컬럼 데이터를 생성 합니다.
    key string - 가져올 값의 key 컬럼명
    val string | number | boolean | null -

    저장할 값

    key 인자 값과 같은 id 속성값을 가진 HTML 요소가 있다면 null 은 빈 문자열 숫자는 숫자 문자열, Boolean 은 "true" 나 "false" 문자열로 변환 되어 저장 되고 대상 HTML 요소가 없다면 원형 그대로 저장 됩니다.

    예제 Examples

    예제 데이터

    
var data = [{
    "name": "Dean Stanley",
    "age": 26,
    "gender": "male",
    "email": "deanstanley@zentia.com",
    "registered": "20140727",
}]

    1. 폼에 데이터 바인드

    폼을 이루는 요소는 TABLE 이든 DIV 든 상관없습니다. 지정한 context 안에 데이터의 키(컬럼)명과 같은 id 속성을 가진 요소가 있다면 해당 요소에 값을 바인드 합니다.

    옵션을 오브젝트형이 아닌 string 형으로 context 만 지정하면 array[json object] 의 0 번째 행 데이터를 자동으로 바인드 합니다.

    
<table id="formExample10">
	<tr>
		<th><label for="name">name</label></th>
		<td><input id="name" type="text" /></td>
	</tr>
	<tr>
		<th><label for="age">age</label></th>
		<td><input id="age" type="text" /></td>
	</tr>
	<tr>
		<th><label for="email">email</label></th>
		<td><input id="email" type="text" /></td>
	</tr>
	<tr>
		<th>registered</th>
		<td id="registered"></td>
	</tr>
</table>

<script type="text/javascript">
	N(data).form("#formExample10").bind();

	/* 위 구문은 다음과 같습니다.
		N(data).form({
			context: "#formExample10",
			row: 0
		}).bind();
	*/
</script>

    2. 폼에 데이터 바인드 및 데이터 포메팅(N.formatter)  >>>  저장 메시지 출력(N.alert) 후 폼 일괄 검증(N.validator)  >>>  저장(N.comm)

    
<table id="formExample20">
	<tr>
		<th><label for="name">name</label></th>
		<td><input id="name" type="text" /></td>
	</tr>
	<tr>
		<th><label for="age">age</label></th>
		<td><input id="age" type="text" data-validate='[ ["required"], ["integer"] ]' /></td> <!-- 필수입력과 숫자만 입력 체크 -->
	</tr>
	<tr>
		<th><label for="email">email</label></th>
		<td><input id="email" type="text" data-validate='[ ["email"] ]' /></td> <!-- 이메일 주소 체크 -->
	</tr>
	<tr>
		<th><label for="registered">registered</label></th>
		<td><input id="registered" type="text" data-format='[ ["date", 8, "date"] ]' /></td> <!-- 날자포멧 및 데이트피커 적용 -->
	</tr>
</table>
<button id="btnFormExample02">저장</button>

<script type="text/javascript">
	var form = N(data).form("#formExample20").bind();

	N("#btnFormExample02").click(function() {
		N(window).alert({ // 1. confirm 메시지 출력
			msg : "저장 하겠습니까?",
			confirm : true,
			onOk : function() {
				if(form.validate()) { // 2. 폼에 입력된 데이터 검증
					N(form.data()).comm("save.json").submit(); // 3. 폼의 데이터를 저장
				}
			}
		});
	});
</script>

    3. 조회 블록(Block)에 N.form 을 적용 하여 검색을 간단하게 처리(신규입력 및 신규입력 데이터 생성)

    
<table id="formExample30">
	<tr>
		<th><label for="name">name</label></th>
		<td><input id="name" type="text" data-validate='[ ["required"] ]' /></td> <!-- 필수 조회조건 체크 -->
		<th><label for="age">age</label></th>
		<td><input id="age" type="text" /></td>
	</tr>
</table>
<button id="btnFormExample30">조회</button>

<script type="text/javascript">
	//add 메서드를 호출하면 id 속성을 가지고 있는 입력요소를 기반으로 { "name": "", "age": "" } 와 같은 데이터가 생성됨.
	var form = N(data).form("#formExample30").add();

	N("#btnFormExample30").click(function() {
		if(form.validate()) { // 1. 조회 블록에 입력된 데이터 검증
			N(form.data()).comm("retrieve.json").submit(); // 2. 폼의 데이터를 조건으로 하여 조회
		}
	});
</script>

    4. N.select 와 N.form 연동

    N.select 와 N.form을 연동하여 사용할 때는 반드시 N.select 를 먼저 초기화 하고 N.form 을 그다음에 초기화 해야 합니다.

    
<table id="formExample40">
	<tr>
		<th><label for="name">name</label></th>
		<td><input id="name" type="text" /></td>
	</tr>
	<tr>
		<th><label for="age">age</label></th>
		<td><input id="age" type="text" /></td>
	</tr>
	<tr>
		<th><label for="gender">gender</label></th>
		<td><input id="gender" type="radio" /></td>
	</tr>
	<tr>
		<th><label for="email">email</label></th>
		<td><input id="email" type="text" /></td>
	</tr>
	<tr>
		<th>registered</th>
		<td id="registered"></td>
	</tr>
</table>

<script type="text/javascript">
	// 1. gender 라디오 셀렉트 생성
	N([{
     	"key" : "male",
     	"val" : "male"
     },{
     	"key" : "female",
     	"val" : "female"
    }]).select({
    	context : "#formExample40 #gender",
    	key : "key",
    	val : "val"
    })

    // 2. 폼 바인드
	N(data).form("#formExample40").bind();
</script>

    5. N.select 를 사용하지 않고 radio, checkbox 입력요소를 N.form 과 연동

    radio 와 checkbox 입력요소의 첫번째 요소에 name 속성값과 같은 id 속성값을 반드시 표기 해야 합니다.

    
<div class="radio">
    <label><input id="age" name="age" value="10"><span>10대</span></label>
    <label><input name="age" value="20"><span>20대</span></label>
    <label><input name="age" value="30"><span>30대</span></label>
</div>
<div class="checkbox">
    <label><input id="color" name="color" value="red"><span>빨간색</span></label>
    <label><input name="color" value="blue"><span>파란색</span></label>
    <label><input name="color" value="green"><span>녹색</span></label>
</div>

<script type="text/javascript">
    var data = {
        age : 10,
        color : ["red", "green"]
    };
    // 1. 폼 바인드
    N(data).form("#formExample40").bind();
</script>
