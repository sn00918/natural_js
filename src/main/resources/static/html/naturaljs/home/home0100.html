Natural-JS

Natural-JS 는 ERP, CRM 등의 기업용 웹 어플리케이션 UI를 쉽고 빠르게 구현할 수 있는 Javascript 프레임워크 라이브러리 입니다.

Natural-JS is a Javascript framework library that can be easily and quickly development the enterprise web application UI such as ERP, CRM, etc.

  • 프레임워크 공통 라이브러리 제공

    Provide framework common libraries

  • 복잡한 웹 UI 환경을 구조화, 단순화 시켜주는 아키텍처 프레임워크 제공

    Provides an architectural framework for structuring and simplifying complex Web UI environments

  • Data 처리 및 데이터 동기화와 관련 된 확장 모듈 제공

    Provides extension modules related to data processing and data synchronization

  • HTML 기반의 다양한 UI 컴포넌트 제공

    Provides various UI components based on HTML

Natural-JS 의 특징

Features of Natural-JS

CVC 아키텍처 패턴 기반의 UI 아키텍처와 이를 지원하는 라이브러리를 제공 합니다.

Provides the CVC architecture pattern based architecture and libraries for support to UI architecture

  • 개발영역과 디자인영역을 완벽하게 분리하여 서버 개발자와 UI 개발자, 또는 스크립트 개발자와 디자이너간의 업무 분쟁을 최소화 할 수 있습니다. 또한 이로 인해 서버 개발 조직과 UI 개발 조직을 분리하여 웹 어플리케이션 개발 프로젝트를 진행 할 수 있습니다.
  • It can completely separate the development and design areas and minimize disputes between server side developers and UI side developer or script developer and designer. In addition, As a result, that may separate the server side and UI side development organization during to do web application development projects.
  • 여러개의 블록페이지(View와 Controller로 구성된 단위 페이지)들을 조합하여 하나의 페이지로 완성할 수 있습니다. 이때 스크립트간, 요소간 Scope 을 완벽하게 분리 해 줍니다.
  • You can combine multiple block pages (page Units consisting of View and Controller) to complete a single page. at this time, it helps ensure full separation between script and elements scope.
  • 단위 페이지별 개발영역의 틀을 정의 할 수 있는 Controller(N.cont) 라이브러리를 제공 하고 Controller 객체의 사용자 정의 메서드들에 대해서 AOP(Aspect-Oriented Programming) 를 적용 할 수 있습니다.
  • Provides the Controller(N.cont) library that can be help to define the frame of the development area per block page and you can apply the AOP(Aspect-Oriented Programming) to methods of Controller's Object
  • UI(클라이언트 웹 브라우저) 와 서버간 데이터 송수신 역할을 담당하는 Communicator(N.comm) 라이브러리를 제공하고 Communicator 로 통신하는 모든 요청과 응답 사이에 공통 실행 로직 이나 공통 데이터 처리기를 적용 할 수 있는 Communication Filter 기능을 제공 합니다.
  • Provides the Communicator(N.comm) library that can be transmitting and receiving data between the UI(Client Web Browser) side and server side and provides a Communication Filter that can apply common execution logic or common data handlers between all requests and responses communicating by Communicator.

데이터의 조작과 가공, 입력된 데이터의 검증을 편리하게 할 수 있는 라이브러리를 제공 합니다.

Provides the libraries for easy control that supports to manipulation and processing of the data, verification of the input data

  • 데이터를 쉽게 서식화(Format) 할 수있는 Formatter 라이브러리를 제공합니다.
  • Provides a Formatter library that can easily format data.
  • 데이터를 쉽게 유효성 검증(Validation) 할 수 있는 Validator 라이브러리를 제공 합니다.
  • Provides a Validator library that can easily validate data.
  • JSON 객체 배열 데이터를 조작 하거나 가공 할 수 있는 filter, sort 등의 라이브러리를 제공 합니다.
  • Provides the filter, sort and etc. libraries for manipulating and processing the JSON Object Array data.

Rich-UI를 표현하기 위한 다양한 UI 컴포넌트를 제공 합니다.

Provides the various UI-Components for express the Rich-UI.

  • 지정한 요소에 데이터 바인드를 손쉽게 할 수 있는 Grid, List, Form, Select, Pagination, Tree 컴포넌트를 제공 합니다.
  • Provides the Grid, List, Form, Select, Pagination and Tree components that can be easily bind the data to the selected elements
  • Rich UI를 쉽게 구현할 수 있도록 해 주는 Alert, Popup, Tab, Button, Datepicker, Notify, Documents 컴포넌트를 제공 합니다.
  • Provides the Alert, Popup, Tab, Button, Datepicker, Notify and Documents components that can be easily development for Rich-UI
  • 조회하거나 생성한 데이터와 HTML 요소의 값을 따로 관리하고 이를 동기화 시켜 줍니다.
  • Queried or generated data and HTML elements to be separately managed and these are synchronized
  • 각각의 UI 컴포넌트들은 실시간으로 데이터가 동기화(양방향 데이터 바인딩)되고 상호작용을 합니다.
  • Data of the each UI-Components are synchronized on real time(Two way databinding) and that is interacted each other.
  • 각각의 컴포넌트는 퍼블리싱된 HTML 템플릿의 모양과 디자인을 최대한 유지시켜 줍니다.

    Grid, Form 등의 컴포넌트는 자체 스타일을 갖고 있지 않습니다. 예를 들어, Grid 컴포넌트를 사용하는 경우, 페이지 내 table 태그에 대한 스타일이 공통 css나 직접 css를 지정하여 적용되어 있다고 하면 Grid 컴포넌트 적용 시 정의된 모양을 최대한 유지 시켜 컴포넌트 화 해 줌으로서 컴포넌트 디자인을 사이트 디자인 컨셉에 간편하게 적용할 수 있습니다.

  • Each components are keeps most the shape and graphic design of be published HTML templete

    Components such as Grid and Form do not have their own style. For example, if you are use the Grid component, If the styles for the table tags in the page are applied by specifying common css or direct css, You can easily apply component design to site design concepts by maintaining the defined table's style when applying the Grid component

모바일 브라우저(iOS Safari, iOS UIWebView, Android Browser, Android Chrome, Android WebView)와 터치 디바이스를 지원 하고 모바일 하이브리드 앱과의 연동을 지원 합니다.

Supports mobile browsers (iOS Safari, iOS UIWebView, Android Browser, Android Chrome, Android WebView) and touch devices and it supports interworking with mobile hybrid applications.
